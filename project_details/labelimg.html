New LabelImg Tool.2022

The goals of this project were labeling mobile application screenshots easily and faster to make a big dataset, and training that dataset after augmentation. Several new buttons are added to the LabelImg tool to make Auto data labeling, augmentation, and training tools. Every AI engineer knows that data labeling requires much time. Therefore, I have implemented an auto-labeling tool. Figure 1 is the working process of Auto Labeling tools. To take screenshots from a Mobile Device, the Screenshot button should be touched and the LabelImg tool posts that screenshot to the Pre-Trained YoloV5 detection model (Multiple-object detection), and then API returns Labels and Bounding box values of all objects in the posted screenshot. After that Label and Bounding boxes will appear on the screenshot in LabelImg tool. However, a developer must check output values and if the Detection model doesn't find a necessary object, the developer labels it, and if the Detection model finds an unnecessary object, the developer deletes it (Video 1). The Data Augmentation button increases the dataset size by resizing, rotation, and so on. And, if the Training button is touched, a dataset will be trained using YoloV5.
