Automation and Rule-Based Testing2021-2023

To save mobile app developers' time, Automation and Rule-Based testing models. To implement the Automation testing tool, I used DroidBot open-source codes changed almost all parts, and added many new features. For example, the Droidbot automation model didn't support IOS devices and has not been connected with AI models and so on. However, the new Automation Testing model supports IOS and Android devices, however Rule-based Testing tool only supports Android devices, but IOS support will be added Rule-based testing tool. Several Object and Text detection models are connected with the Automation tool. They are Yolov5 multi-object detection (to find all objects), Loading detection and classification (to find loading objects), and Search Detection (to find search icons&buttons), EasyOCR and Tesseract Text detection models. WebDriveAgent was used to interact with IOS devices. ADB and UI Automators are utilized to control Android devices. Video 1 is Real-time automation testing Sinla Application on IOS and Android devices using Automation tool.
To implement Rule-based testing tool, several new buttons are added to the LabelImg tool, and it is connected with the Yolov5 multi-object detection model and UI Automator tool to find labels and bounding boxes of objects. The Screenshot button takes a screenshot from an Android device, and the Select button writes the selected object's label and index into an Excel file, after that clicks the selected object and takes a new screenshot again. So, using the Select button, a test cases.xls file will be created. If the Play button is touched, Rule-based testing will be started. Mobile device touches all objects written in cases.xlx order by order as shown in Video 2.
